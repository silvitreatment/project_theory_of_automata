# Отчёт по лабораторной работе
## Теория автоматов — Распознающий КА и поиск цепочек языка

---

## 1. Задание

Разработать распознающий конечный автомат (КА) для цепочек регулярного языка и реализовать его в виде подпрограммы. Разработать программу поиска цепочек этого языка в последовательности символов.

**Язык:** строка символов из алфавита `{+, -, a, b}`, начинающаяся с префикса `+` и заканчивающаяся суффиксом `+`, между которыми расположена последовательность символов `a, b` (она может быть и пустой), в которой любая пара одинаковых символов разделяется символом `-`.

**Важно:** символ `-` допустим **только** между одинаковыми символами. Между разными символами (`a` и `b`) дефис стоять не должен.

**Примеры допустимых цепочек:** `++`, `+a+`, `+ab+`, `+a-a+`, `+b-b+`, `+abab+`, `+a-a-ab-b-bab-b+`

**Примеры НЕ принадлежащих языку:** `+aa+` (нет `-` между одинаковыми), `+a-b+` (дефис между разными), `+-a+` (начинается с `-`), `+a-+` (заканчивается на `-`)

## 2. Регулярное выражение (РВ), задающее язык

РВ описывает множество **допустимых цепочек языка** (не входную последовательность).
Входная последовательность — произвольная строка из алфавита `{+, -, a, b}`,
в которой программа ищет подстроки, принадлежащие языку.

```
+( (a(-a)*b(-b)*)*(a(-a)*)? | (b(-b)*a(-a)*)*(b(-b)*)? )?+
```

Расшифровка:
- `+` — открывающий символ (префикс)
- Внутренняя часть (необязательная) состоит из **серий** одного символа:
  - **Серия `a`:** `a(-a)*` — один или более символов `a`, разделённых `-`
  - **Серия `b`:** `b(-b)*` — один или более символов `b`, разделённых `-`
  - Серии разных символов чередуются **без** разделителя между ними
- Два варианта через `|`: начинается с серии `a` или с серии `b`
- `+` — закрывающий символ (суффикс)

## 3. Диаграмма переходов КА

```
  q0 ──(+)──► q1
  q0 ──(-,a,b)──► qE

  q1 ──(+)──► q5  [принято: ++]
  q1 ──(a)──► q2
  q1 ──(b)──► q3
  q1 ──(-)──► qE

  q2 ──(+)──► q5  [принято]
  q2 ──(b)──► q3  [смена символа без -]
  q2 ──(-)──► q4a [дефис после a]
  q2 ──(a)──► qE  [aa без разделителя]

  q3 ──(+)──► q5  [принято]
  q3 ──(a)──► q2  [смена символа без -]
  q3 ──(-)──► q4b [дефис после b]
  q3 ──(b)──► qE  [bb без разделителя]

  q4a ──(a)──► q2  [дефис разделил одинаковые a]
  q4a ──(+,-,b)──► qE  [дефис между разными — запрещено]

  q4b ──(b)──► q3  [дефис разделил одинаковые b]
  q4b ──(+,-,a)──► qE  [дефис между разными — запрещено]

  q5 ──(все)──► qE  [принимающее, нет переходов]
  qE ──(все)──► qE  [тупиковое]
```

## 4. Таблица переходов КА

| Состояние | `+` | `-` | `a` | `b` | Описание |
|-----------|-----|-----|-----|-----|----------|
| **q0** | q1 | qE | qE | qE | Начальное |
| **q1** | q5 | qE | q2 | q3 | Прочитан открывающий `+` |
| **q2** | q5 | q4a | qE | q3 | Последний внутренний символ — `a` |
| **q3** | q5 | q4b | q2 | qE | Последний внутренний символ — `b` |
| **q4a** | qE | qE | q2 | qE | Прочитан `-` после `a` (ожидаем только `a`) |
| **q4b** | qE | qE | qE | q3 | Прочитан `-` после `b` (ожидаем только `b`) |
| **q5** | qE | qE | qE | qE | **Принимающее** (прочитан закрывающий `+`) |
| **qE** | qE | qE | qE | qE | Тупиковое (ошибка) |

- **Начальное состояние:** q0
- **Принимающее состояние:** {q5}
- **Количество состояний:** 8 (q0, q1, q2, q3, q4a, q4b, q5, qE)

## 5. Листинги программ

### 5.1 Подпрограмма-распознаватель КА (`solution.py`)

```python
# Алфавит языка
ALPHABET: set[Symbol] = {"+", "-", "a", "b"}

# Состояния КА
Q0: State = "q0"    # начальное состояние
Q1: State = "q1"    # прочитан открывающий '+'
Q2: State = "q2"    # последний внутренний символ — 'a'
Q3: State = "q3"    # последний внутренний символ — 'b'
Q4A: State = "q4a"  # прочитан '-' после 'a' (ожидается только 'a')
Q4B: State = "q4b"  # прочитан '-' после 'b' (ожидается только 'b')
Q5: State = "q5"    # принимающее (прочитан закрывающий '+')
QE: State = "qE"    # состояние ошибки (тупиковое)

# Таблица переходов ДКА
TRANSITION: TransitionTable = {
    Q0:  {"+": Q1,  "-": QE,  "a": QE, "b": QE},
    Q1:  {"+": Q5,  "-": QE,  "a": Q2, "b": Q3},
    Q2:  {"+": Q5,  "-": Q4A, "a": QE, "b": Q3},
    Q3:  {"+": Q5,  "-": Q4B, "a": Q2, "b": QE},
    Q4A: {"+": QE,  "-": QE,  "a": Q2, "b": QE},
    Q4B: {"+": QE,  "-": QE,  "a": QE, "b": Q3},
    Q5:  {"+": QE,  "-": QE,  "a": QE, "b": QE},
    QE:  {"+": QE,  "-": QE,  "a": QE, "b": QE},
}

def dfa_recognize(chain: str) -> bool:
    """Проверяет, принадлежит ли цепочка языку через ДКА."""
    state: State = Q0
    for symbol in chain:
        if symbol not in ALPHABET:
            return False
        state = TRANSITION[state][symbol]
        if state == QE:
            return False
    return state == ACCEPT_STATE
```

### 5.2 Программа поиска цепочек (`solution.py`)

```python
def find_chains(sequence: str) -> list[tuple[int, str]]:
    """
    Ищет все цепочки языка в последовательности символов.
    Использует подпрограмму dfa_recognize для проверки кандидатов.
    """
    results: list[tuple[int, str]] = []
    n: int = len(sequence)
    i: int = 0

    while i < n:
        if sequence[i] != "+":
            i += 1
            continue

        last_accept_end: int = -1
        for j in range(i + 1, n + 1):
            if sequence[j - 1] not in ALPHABET:
                break
            candidate: str = sequence[i:j]
            if dfa_recognize(candidate):
                last_accept_end = j

        if last_accept_end != -1:
            chain: str = sequence[i:last_accept_end]
            results.append((i + 1, chain))
            # Закрывающий '+' = начало следующей цепочки
            i = last_accept_end - 1
        else:
            i += 1

    return results
```

## 6. Краткое описание работы программ

### Подпрограмма `dfa_recognize`

Реализует детерминированный конечный автомат (ДКА) с 8 состояниями. Принимает строку и посимвольно выполняет переходы по таблице `TRANSITION`. Если символ не из алфавита или автомат попадает в тупиковое состояние `qE` — возвращает `False`. После прочтения всей строки возвращает `True` только если автомат находится в принимающем состоянии `q5`.

**Ключевые правила автомата:**
- Из `q2` (последний был `a`) переход по `a` → `qE` (два одинаковых без `-`)
- Из `q3` (последний был `b`) переход по `b` → `qE` (два одинаковых без `-`)
- Из `q2` переход по `-` → `q4a` (ожидаем **только** `a` — дефис между одинаковыми)
- Из `q3` переход по `-` → `q4b` (ожидаем **только** `b` — дефис между одинаковыми)
- Из `q4a` переход по `b` → `qE` (дефис между разными — запрещено)
- Из `q4b` переход по `a` → `qE` (дефис между разными — запрещено)

### Программа `find_chains`

Сканирует входную последовательность слева направо. Для каждой позиции с символом `+` перебирает все возможные концы подстроки и вызывает `dfa_recognize` для проверки каждого кандидата. Запоминает самую длинную принятую цепочку (жадный поиск). После нахождения цепочки следующий поиск начинается с последнего символа найденной цепочки (закрывающий `+` может быть открывающим следующей).
