# Отчёт по лабораторной работе
## Теория автоматов — Распознающий КА и поиск цепочек языка

---

## 1. Задание

Разработать распознающий конечный автомат (КА) для цепочек регулярного языка и реализовать его в виде подпрограммы. Разработать программу поиска цепочек этого языка в последовательности символов.

**Язык:** строка символов из алфавита `{+, -, a, b}`, начинающаяся с префикса `+` и заканчивающаяся суффиксом `+`, между которыми расположена последовательность символов `a, b` (она может быть и пустой), в которой любая пара одинаковых символов разделяется символом `-`.

**Примеры допустимых цепочек:** `+a-a-ab-b-bab-b+`, `++`, `+abab+`, `+a+`, `+b-b+`

---

## 2. Регулярное выражение (РВ), задающее язык

РВ описывает множество **допустимых цепочек языка** (не входную последовательность).
Входная последовательность — произвольная строка из алфавита `{+, -, a, b}`,
в которой программа ищет подстроки, принадлежащие языку.

```
+((a|b)((-?(a|b))*)?)?+
```

Расшифровка:
- `+` — открывающий символ (префикс)
- `((a|b)((-?(a|b))*)?)?` — необязательная внутренняя часть:
  - `(a|b)` — первый символ: `a` или `b`
  - `(-?(a|b))*` — ноль или более продолжений: необязательный `-`, затем `a` или `b`
  - ограничение языка: если два соседних символа одинаковы (`aa` или `bb`), между ними **обязан** стоять `-`; между разными символами `-` необязателен
- `+` — закрывающий символ (суффикс)

**Примеры цепочек языка:** `++`, `+a+`, `+ab+`, `+a-a+`, `+abab+`, `+a-a-ab-b-bab-b+`
**Примеры НЕ принадлежащих языку:** `+aa+` (нет `-` между одинаковыми), `+-a+` (начинается с `-`), `+a-+` (заканчивается на `-`)

---

## 3. Диаграмма переходов КА

```
        +           a           b
  ┌──────────────────────────────────┐
  │                                  ▼
 [q0] ──+──► [q1] ──a──► [q2] ──+──► ((q5))
                │          │ ▲    │
                │          - │    b
                │          │ │    │
                b          ▼ │    ▼
                │         [q4]──a─►[q2]
                │          │
                │          b
                │          │
                ▼          ▼
               [q3] ◄──────┘
                │
                +──► ((q5))
                │
                -──► [q4]
                │
               [qE] (тупиковое)
```

**Полная диаграмма переходов:**

```
  q0 ──(+)──► q1
  q0 ──(-,a,b)──► qE

  q1 ──(+)──► q5  [принято: ++]
  q1 ──(a)──► q2
  q1 ──(b)──► q3
  q1 ──(-)──► qE

  q2 ──(+)──► q5  [принято]
  q2 ──(b)──► q3
  q2 ──(-)──► q4
  q2 ──(a)──► qE  [aa без разделителя]

  q3 ──(+)──► q5  [принято]
  q3 ──(a)──► q2
  q3 ──(-)──► q4
  q3 ──(b)──► qE  [bb без разделителя]

  q4 ──(a)──► q2
  q4 ──(b)──► q3
  q4 ──(+,-)──► qE

  q5 ──(все)──► qE  [принимающее, нет переходов]
  qE ──(все)──► qE  [тупиковое]
```

- **Принимающее состояние:** {q5}

## 5. Листинги программ

### 5.1 Подпрограмма-распознаватель КА (`solution.py`)

```python
# Алфавит языка
ALPHABET = {'+', '-', 'a', 'b'}

# Состояния КА
Q0 = 'q0'  # начальное
Q1 = 'q1'  # прочитан открывающий '+'
Q2 = 'q2'  # последний внутренний символ — 'a'
Q3 = 'q3'  # последний внутренний символ — 'b'
Q4 = 'q4'  # прочитан разделитель '-'
Q5 = 'q5'  # принимающее (прочитан закрывающий '+')
QE = 'qE'  # состояние ошибки

# Таблица переходов: transition[state][symbol] -> next_state
TRANSITION = {
    Q0: {'+': Q1, '-': QE, 'a': QE, 'b': QE},
    Q1: {'+': Q5, '-': QE, 'a': Q2, 'b': Q3},
    Q2: {'+': Q5, '-': Q4, 'a': QE, 'b': Q3},
    Q3: {'+': Q5, '-': Q4, 'a': Q2, 'b': QE},
    Q4: {'+': QE, '-': QE, 'a': Q2, 'b': Q3},
    Q5: {'+': QE, '-': QE, 'a': QE, 'b': QE},
    QE: {'+': QE, '-': QE, 'a': QE, 'b': QE},
}

def dfa_recognize(chain: str) -> bool:
    """
    Подпрограмма-распознаватель: проверяет, принадлежит ли
    цепочка chain языку с помощью детерминированного КА.
    """
    state = Q0  # начинаем в начальном состоянии

    for symbol in chain:
        if symbol not in ALPHABET:
            return False
        state = TRANSITION[state][symbol]
        if state == QE:
            return False

    return state == ACCEPT_STATE
```

### 5.2 Программа поиска цепочек (`solution.py`)

```python
def find_chains(sequence: str) -> list[tuple[int, str]]:
    """
    Ищет все цепочки языка в последовательности символов.
    Для каждой позиции i с символом '+' запускает КА и жадно
    ищет самую длинную принимаемую цепочку.
    Закрывающий '+' считается началом следующей возможной цепочки.
    """
    results = []
    n = len(sequence)
    i = 0

    while i < n:
        if sequence[i] != '+':
            i += 1
            continue

        state = Q0
        last_accept_end = -1

        for j in range(i, n):
            symbol = sequence[j]
            if symbol not in ALPHABET:
                break
            state = TRANSITION[state][symbol]
            if state == QE:
                break
            if state == ACCEPT_STATE:
                last_accept_end = j + 1

        if last_accept_end != -1:
            chain = sequence[i:last_accept_end]
            results.append((i + 1, chain))  # позиция 1-based
            # Закрывающий '+' = начало следующей цепочки
            i = last_accept_end - 1
        else:
            i += 1

    return results
```

## 6. Краткое описание работы программ

### Подпрограмма `dfa_recognize`

Реализует детерминированный конечный автомат (ДКА) с 7 состояниями. Принимает строку и посимвольно выполняет переходы по таблице `TRANSITION`. Если символ не из алфавита или автомат попадает в тупиковое состояние `qE` — возвращает `False`. После прочтения всей строки возвращает `True` только если автомат находится в принимающем состоянии `q5`.

**Ключевые правила автомата:**
- Из `q2` (последний был `a`) переход по `a` → `qE` (два одинаковых без `-`)
- Из `q3` (последний был `b`) переход по `b` → `qE` (два одинаковых без `-`)
- Из `q4` (после `-`) переход по `+` или `-` → `qE` (нельзя заканчивать на `-`)

### Программа `find_chains`

Сканирует входную последовательность слева направо. Для каждой позиции с символом `+` запускает ДКА и жадно ищет самую длинную принимаемую цепочку (продолжает чтение даже после первого принятия). После нахождения цепочки следующий поиск начинается с последнего символа найденной цепочки (закрывающий `+` может быть открывающим следующей).
